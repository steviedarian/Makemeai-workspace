# Customization Guide — Telegram AI Command Bot

How to make this bot yours. Every section targets a specific file you can edit.

---

## 1. Bot Persona — `worker.py`

The `_GENERAL_AGENT_PROMPT` string in `worker.py` defines how your General agent behaves. This is the system prompt appended to every General topic interaction.

**What to change:**

Open `apps/command/worker.py` and find the `_GENERAL_AGENT_PROMPT` variable. Replace it with your own persona:

```python
_GENERAL_AGENT_PROMPT = """\
You are [NAME]'s main Telegram assistant — their General agent for [BUSINESS NAME].
You are a persistent Claude Code agent running on Sonnet. [NAME] chats with you throughout the day,
primarily from their phone. You have full workspace power — files, database, web search, everything.

## Your Role
- Strategic thinking partner and chief of staff
- Data analyst (query data/[your-db].db via sqlite3)
- Quick research via WebSearch
- Task coordinator (when [NAME] needs a fresh agent, tell them to use /new or /new opus)

## Telegram Rules
- Keep responses concise — phone screen, on the go
- Use bold and bullets for readability
- For detailed analysis: send inline first, then offer PDF for long reports
- Be direct and conversational. No corporate fluff.

## Voice Notes
When voice notes are received (transcribed by Whisper): read current context first,
cross-reference with existing data, ask clarifying questions when ambiguous.

## Image Analysis
When photos are sent, they are saved to data/command/photos/ and the file path appears in the message.
Use the Read tool to view the image. Analyze screenshots, charts, documents, handwritten notes, etc.
"""
```

The `_BASE_PROMPT` is used for spawned agents (from `/new`). It is more generic. Customize it the same way if you want a different persona for one-off agents.

## 2. Workspace Context — `CLAUDE.md`

This is the most impactful file. Every agent reads it at the start of every session.

Use the `CLAUDE.md.template` file in this kit as a starting point. Key sections:

- **What This Is** — 2-3 sentences. The agent uses this to orient itself.
- **Key Files** — list every file the agent might need. Be specific with paths.
- **Conventions** — currency, timezone, terminology, formatting rules.
- **Commands** — if you build slash commands, document them here.
- **Workspace Structure** — folder layout so the agent can navigate.

**Pro tips:**
- The more context you put in CLAUDE.md, the smarter every interaction becomes
- Update it as your workspace evolves — stale CLAUDE.md = confused agents
- Keep it under 15K tokens (roughly 10,000 words) — longer files slow down priming
- Use headers and structure — agents parse structured text much better than prose

## 3. Prime Commands — `.claude/commands/`

Prime commands tell agents which files to read when they start.

**`prime-telegram.md`** — used by the General agent and spawned Telegram agents. Keep this SHORT (5-8 files max). Agents need to prime fast because users are waiting on their phone.

**`prime.md`** — used for full Claude Code sessions (non-Telegram). Can be longer and more thorough.

**What to point them at:**

Think about what context an agent needs to be useful in 80% of conversations:
- Your CLAUDE.md (always)
- Your current strategy or roadmap document
- Your latest metrics or KPI file
- Your team roster (if relevant)
- Any reference docs that get used frequently

Example `prime-telegram.md`:
```markdown
# Prime (Telegram)

> Short prime for Telegram agents. Keep acknowledgment brief.

Read these files in order:
1. `CLAUDE.md` — Workspace overview and conventions
2. `context/strategy.md` — Current strategic priorities
3. `context/metrics.md` — Latest business metrics
4. `context/team.md` — Team roster and responsibilities

After reading, respond briefly:
**Primed and ready.** What do you need?
```

## 4. Brand Colors — `chart_style.py`

Charts generated by agents use the color palette defined in `apps/command/chart_style.py`.

**Change the colors:**

```python
# Open apps/command/chart_style.py

COLORS = {
    "primary": "#1a73e8",      # Your primary brand color
    "secondary": "#34a853",     # Secondary color
    "accent1": "#ea4335",       # Accent colors for chart series
    "accent2": "#fbbc04",
    "accent3": "#9334e6",
    "accent4": "#ff6d01",
    "light_bg": "#f8f9ff",      # Light background for chart area
    "grid": "#e8e8e8",          # Grid lines
    "text": "#1a1a1a",          # Axis labels and titles
    "text_muted": "#666666",    # Secondary text
}
```

The `PALETTE` list controls the order of colors used for multi-series charts:
```python
PALETTE = [
    COLORS["primary"], COLORS["secondary"], COLORS["accent1"],
    COLORS["accent2"], COLORS["accent3"], COLORS["accent4"],
]
```

## 7. Adding Special Topics

You can register specific forum topics for specific personas or purposes. The pattern:

1. **Create a topic** in your Telegram group (right-click or long-press --> "New Topic")
2. **Get the topic ID** by sending a message in the topic and checking the bot logs for `message_thread_id`
3. **Add a config entry** in `.env`:
   ```
   MY_SPECIAL_TOPIC_ID=12345
   ```
4. **Load it in config.py:**
   ```python
   my_special_topic_id: int | None
   # In load_config():
   my_special_topic_id=int(v) if (v := os.getenv("MY_SPECIAL_TOPIC_ID", "").strip()) else None,
   ```
5. **Add routing in bot.py** (in `_flush_message_buffer`, before the General catch-all):
   ```python
   elif topic_id and _config.my_special_topic_id and topic_id == _config.my_special_topic_id:
       await _orchestrator.handle_my_special_topic(last_message, text_override=combined_text, photos=all_photos or None)
   ```
6. **Add the handler in orchestrator.py** with a custom system prompt for that topic's purpose.

**Examples of special topics:**
- **Inbox topic** — for GTD inbox processing with a specialized prompt
- **Content topic** — for content ideation with access to your content pipeline
- **Research topic** — for deep research with a prompt that emphasizes web search
- **Data topic** — for analytics with a prompt that knows your database schema

## 8. Adding Database Queries

Agents can query any SQLite database in your workspace. The key is telling them about it in their system prompt.

**Step 1:** Put your SQLite database somewhere in the workspace (e.g., `data/my-database.db`)

**Step 2:** Add database info to your `CLAUDE.md`:
```markdown
## Database
SQLite database at `data/my-database.db`. Key tables:
- `customers` — id, name, email, plan, created_at
- `revenue` — id, customer_id, amount, date
- `metrics` — id, metric_name, value, date

Example queries:
- Monthly revenue: `SELECT strftime('%Y-%m', date) as month, SUM(amount) FROM revenue GROUP BY month`
- Active customers: `SELECT COUNT(*) FROM customers WHERE plan != 'cancelled'`
```

**Step 3:** Add database context to the General agent prompt in `worker.py`:
```python
_GENERAL_AGENT_PROMPT = """\
...
## Database
You have a SQLite database at data/my-database.db.
Query it using: sqlite3 data/my-database.db "SELECT ..."
Key tables: customers, revenue, metrics.
...
"""
```

Agents will use the Bash tool to run `sqlite3` queries directly. They can also write Python scripts that query the database for more complex analysis.
